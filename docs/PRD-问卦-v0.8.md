## 1. Executive Summary

**问卦 v0.8** 新增三个功能：姓名+生辰交叉起卦（名字配你的命合不合）、小六壬掌诀占卜（手掌上三秒出结果）、五格姓名学（你的名字能打多少分）。v0.8 之后问卦将拥有六种起卦 + 三种高级分析，正式从"多模式算卦工具"升级为"易学百宝箱"。

## 2. Problem Statement

### 用户痛点

**交叉起卦**：
- 现在的"姓名起卦"只看名字本身的好坏，"生辰起卦"只看命格走势
- 用户真正想问的是"我这个名字和我的命配不配"——需要一个交叉分析

**小六壬**：
- 现有所有模式都需要理解卦象，对易经小白门槛高
- 用户有"快速出门问事"的需求，不想等六次掷爻也不想研究卦辞

**五格姓名学**：
- "你的名字能打多少分"是社交传播的天然钩子
- 现有姓名起卦只有卦象和体用生克，缺数值化的吉凶评分

### 业务影响

- 小六壬是差异化极强的功能（同赛道几乎没有），极易传播
- 五格姓名学是社交裂变利器（"我名字85分！你的呢？"）
- 交叉起卦把两个已有模式的用户体验拉到新高度

## 4. User Personas

### Persona H（新增）：实用主义者 — 阿杰
- **年龄**: 25-35 岁，不信玄学但相信大数据
- **场景**: 朋友分享了一个"名字打分"链接，点进去输入名字→看到 85 分→转发
- **需求**: 快、有数字、能分享

### Persona I（新增）：速食算卦族 — 小安
- **年龄**: 20-30 岁，出门前想快速问一事
- **场景**: 打开问卦→点小六壬→输当前时间→拇指在手掌上点三下→"速喜·小吉"→放心出门
- **需求**: 3 秒出结果，6 个结果就够，不需要理解卦

## 5. User Stories

### US-1: 姓名+生辰交叉起卦（P0）

```
Feature: 姓名与生辰双因子起卦

  Scenario: 用户获得交叉卦
    Given 用户进入"交叉起卦"界面
    When 输入姓名并选择出生年月日时
    And 点击"交叉起卦"
    Then 系统取姓氏笔画+年支数→取模得上卦
    And 取名字总笔画+月份→取模得下卦
    And 取日期+时辰→取模得动爻
    And 通过梅花引擎生成本卦+互卦+变卦+体用生克
    And 结果区展示：本卦+互卦+变卦+体用生克+交叉解读

  Scenario: 交叉解读说明
    Given 交叉卦已生成
    When 体用生克结果得出
    Then 如果体用相生: "你的名字五行×，生辰五行×，两者相生。这个名字和你很配 ✅"
    And 如果体用相克: "你的名字五行×，生辰五行×，两者相克。这个名字可能不太适合你 ⚠️"

  Acceptance Criteria:
  - [ ] 复用 stroke-data.js + lunar.js + 梅花引擎
  - [ ] 上卦=(姓氏笔画+年支数)%8, 下卦=(名字总笔画+月份)%8, 动爻=(日期+时辰)%6
  - [ ] 互卦和变卦走标准梅花取爻法（不由生辰数字干预）
  - [ ] 交叉解读文案 ~30 字/条（5 级生克各一条）
```

### US-2: 小六壬掌诀占卜（P0）

```
Feature: 小六壬手掌占卜

  Scenario: 用户以当前时间占卜
    Given 用户进入小六壬界面
    When 点击"用当前时间"或手动输入农历月/日/时
    And 点击"掐指一算"
    Then 系统从寅位(大安)起正月
    And 顺时针数到目标月份→落位后从该位起初一
    And 数到目标日子→落位后再从该位起子时
    And 数到目标时辰→最终落位即掌诀结果
    And 展示结果：大安/留连/速喜/赤口/小吉/空亡 + 解读

  Scenario: 手掌动画
    Given 用户点击"掐指一算"
    When 推算进行中
    Then 左手手掌 SVG/CSS 动画展示拇指在三步点过六个位置
    And 最终位置高亮闪烁

  Acceptance Criteria:
  - [ ] 六个掌诀名+吉凶等级+解读文案（各 ~40 字）
  - [ ] 月日时三推算法正确（对照标准小六壬推算法）
  - [ ] 手掌动画可用纯 CSS + SVG 实现
```

### US-3: 五格姓名学评分（P0）

```
Feature: 五格数理姓名评分

  Scenario: 用户查看姓名五格评分
    Given 用户在姓名起卦结果页面
    When 页面渲染完成
    Then 在卦象解读下方展示五格数理卡片:
      天格: ×× (×画, 大吉/凶/半吉)
      人格: ×× (×画, 大吉/凶/半吉) ← 核心
      地格: ×× (×画, 大吉/凶/半吉)
      总格: ×× (×画, 大吉/凶/半吉)
      外格: ×× (×画, 大吉/凶/半吉)
    And 底部展示三才配置(天→人→地五行相生/克)
    And 综合评分(五格加权得分)

  Acceptance Criteria:
  - [ ] 天格=姓笔画+1(单姓)/姓笔画和(复姓)
  - [ ] 人格=姓末字+名首字
  - [ ] 地格=名全部笔画(单名+1)
  - [ ] 总格=全部笔画
  - [ ] 外格=总格-人格+1
  - [ ] 81 数理吉凶表完整内嵌
  - [ ] 大吉/凶/半吉 用颜色区分
```

## 8. Technical Considerations

### 8.1 小六壬核心算法

```javascript
// six-yao.js 或直接在 app.js 中
const LIUREN_POSITIONS = ['大安','留连','速喜','赤口','小吉','空亡'];

function xiaoliuren(month, day, hour) {
    // 正月从寅位(大安=index0)起, 顺数 month 个月
    let pos = (month - 1) % 6;  // 月落位
    // 从月落位起初一, 顺数 day 天
    pos = (pos + day - 1) % 6;  // 日落位
    // 从日落位起子时, 顺数 hour 个时辰
    pos = (pos + hour - 1) % 6; // 时落位 = 最终结果
    return LIUREN_POSITIONS[pos];
}
```

### 8.2 五格计算（复用 stroke-data.js）

```javascript
function calcWuge(parsed) {
    const surname = surnameStrokes(parsed);
    const given = givenStrokes(parsed);
    const total = surname + given;
    return {
        tian: parsed.type.startsWith('compound') ? surname : surname + 1,
        ren:  parsed.type.endsWith('single')
            ? surname + (given || 0) + 1
            : surname + firstGivenStroke(parsed),
        di:  parsed.type.endsWith('single') ? (given || 0) + 1 : given,
        zong: total,
        wai: total - renGe + 1
    };
}
```

## 12. Open Questions

| # | 问题 | 讨论 |
|---|------|------|
| Q1 | 五格姓名学是独立标签还是增强现有姓名模式？ | 增强现有。五格 + 姓名卦 = 完整姓名报告（上半卦象、下半五格）。独立标签会让用户困惑"姓名起卦/姓名评分有什么区别" |
